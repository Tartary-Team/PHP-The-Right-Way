Код стилі бойынша нұсқаулық

PHP қауымдастығы үлкен және алуан түрлі, ол көптеген кітапханалардан, құрылымдардан және компоненттерден тұрады. Әдетте, PHP әзірлеушілері олардың бірнешеуін таңдап, бір жобаға біріктіреді. Әзірлеушілерге өз жобалары үшін әртүрлі кітапханаларды араластыру және сәйкестендіру оңай болуы үшін PHP кодының жалпы код стилін (мүмкіндігінше) ұстануы маңызды.

Фреймворктармен өзара әрекеттесу тобы бірқатар стильдік ұсыныстарды ұсынды және мақұлдады. Олардың барлығы код стиліне жатпайды, бірақ олар PSR - 1, PSR-12, PSR-4 және PER Coding Style. Бұл ұсыныстар Drupal, Zend, Symfony, Laravel, CakePHP, phpBB, AWS SDK, FuelPHP, Lithium және т. б. сияқты көптеген жобалар ұстанатын ережелер жиынтығы ғана. біз асырап аламыз. Сіз оларды өз жобаларыңыз үшін пайдалана аласыз немесе жеке стиліңізді пайдалануды жалғастыра аласыз.

Ең дұрысы, сіз белгілі стандартқа сәйкес келетін PHP кодын жазуыңыз керек. Бұл кез келген PSR комбинациясы немесе PEAR немесе Zend әзірлеген кодтау стандарттарының бірі болуы мүмкін. Бұл дегеніміз, басқа әзірлеушілер сіздің кодты оңай оқи алады және онымен жұмыс істей алады, ал компоненттерді жүзеге асыратын қосымшалар үшінші тарап кодымен жұмыс істегенде де дәйектілікті қамтамасыз ете алады.

  • PSR - 1 туралы оқыңыз
  • PSR - 12 туралы оқыңыз
  • PSR-4 туралы оқыңыз
  • Әр кодтау стилі туралы оқыңыз
  • PEAR кодтау стандарттары туралы оқыңыз
  • Symfony кодтау стандарттары туралы оқыңыз

Сіз PHP_CodeSniffer көмегімен кодты осы ұсыныстардың кез-келгеніне сәйкестігін тексере аласыз, ал нақты уақыт режимінде кері байланыс алу үшін Sublime Text сияқты мәтіндік редакторларға арналған плагиндер.

Төмендегі құралдардың бірін пайдаланып кодтың орналасуын автоматты түрде түзетуге болады:

  • Олардың бірі-өте жақсы тексерілген кодтық базасы бар PHP Coding standards Fixer.
  • Сонымен қатар, PHP_CodeSniffer жиынтығына кіретін PHP кодын жақсарту және түзету құралын кодты сәйкесінше теңшеу үшін пайдалануға болады.

Phpcs-ті қабықтан қолмен іске қосуға болады:

phpcs -sw --standard=PSR1 file.php

Ол қателерді көрсетеді және оларды қалай түзетуге болатынын сипаттайды. Бұл пәрменді git ілгегіне қосу пайдалы болуы мүмкін. Осылайша, таңдалған стандартты бұзушылықтары бар филиалдар репозиторийге осы бұзушылықтар түзетілгенге дейін жете алмайды.

Егер сізде PHP_CodeSniffer болса, онда сіз PHP code Beautifier және Fixer көмегімен есеп беретін кодты орналастыру мәселелерін автоматты түрде түзете аласыз.

phpcbf -w --standard=PSR1 file.php

Тағы бір нұсқа - PHP кодтау стандарттарын түзету құралын пайдалану. Бұл код құрылымында қандай қателер оларды түзетпес бұрын болғанын көрсетеді.

php-cs-fixer fix -v --rules=@PSR1 file.php

Барлық таңбалар атаулары мен Код инфрақұрылымы үшін ағылшын тіліне артықшылық беріледі. Түсініктемелерді кодтық базада жұмыс істей алатын барлық қазіргі және болашақ мүшелер оңай оқитын кез-келген тілде жазуға болады.

Сонымен, таза PHP кодын жазу үшін жақсы қосымша ресурс-бұл Clean Code PHP.
