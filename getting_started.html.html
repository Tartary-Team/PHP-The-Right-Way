Бастау

Ағымдағы тұрақты нұсқаны қолданыңыз (8.2)

Егер сіз PHP - мен жұмыс істей бастасаңыз, ағымдағы PHP 8.2 тұрақты нұсқасынан бастаңыз. PHP 8.x ескі 7 нұсқасымен салыстырғанда көптеген жаңа мүмкіндіктерді қосады.x және 5.x нұсқалары. Қозғалтқыш негізінен қайта жазылды және PHP қазір ескі нұсқаларға қарағанда тезірек жұмыс істейді. PHP 8 - бұл тілдің негізгі жаңартуы және көптеген жаңа мүмкіндіктер мен оңтайландырулардан тұрады.

Сіз соңғы тұрақты нұсқаға жылдам жаңартуға тырысуыңыз керек - PHP 7.4 мерзімі өтіп кеткен. Жаңарту қарапайым, өйткені PHP 8.0, PHP 8.1, PHP 8.2 кері үйлесімділік бұзылыстары көп емес. Егер сіз функцияның қай нұсқада екеніне сенімді болмасаңыз, PHP құжаттамасын веб-сайттан тексере аласыз php.net.

Кірістірілген веб-сервер

PHP 5.4 немесе одан кейінгі нұсқаларында сіз толық веб-серверді орнатпай және орнатпай-ақ PHP-ді үйренуді бастай аласыз. Серверді іске қосу үшін жобаның түбірлік каталогындағы терминалдан келесі пәрменді іске қосыңыз:

 > php -S localhost:8000

Кірістірілген пәрмен жолы веб-сервері туралы біліңіз

MacOS орнату

macOS PHP-мен бірге келеді, бірақ ол әдетте соңғы тұрақты нұсқадан сәл артта қалады. PHP-дің соңғы нұсқасын macOS-қа орнатудың бірнеше әдісі бар.

Homebrew арқылы PHP орнатыңыз

Homebrew-бұл PHP және әртүрлі кеңейтімдерді оңай орнатуға көмектесетін mac OS пакет менеджері. Homebrew core репозиторийі PHP 7.4, 8.0, 8.1 және PHP 8.2 үшін “формулаларды” ұсынады. Осы пәрменмен соңғы нұсқаны орнатыңыз:

brew install php@8.2

Path айнымалысын өзгерту арқылы Homebrew PHP нұсқалары арасында ауысуға болады. Сонымен қатар, PHP нұсқаларын автоматты түрде ауыстыру үшін brew-php-switch пайдалануға болады.

Сондай-ақ, қажетті нұсқаны өшіру және байланыстыру арқылы PHP нұсқаларын қолмен ауыстыруға болады:

brew unlink php
brew link --overwrite php@8.1

brew unlink php
brew link --overwrite php@8.2

MACPORTS арқылы PHP орнатыңыз

MacPorts жобасы-бұл macOS операциялық жүйесіндегі X11 немесе Aqua пәрмен жолына негізделген ашық бастапқы бағдарламалық құралды құрастыру, орнату және жаңарту үшін қолдануға оңай жүйені әзірлеу үшін ашық бастапқы қауымдастық бастамасы.

MacPorts алдын-ала құрастырылған екілік файлдарды қолдайды, сондықтан әр тәуелділікті бастапқы мұрағаттық файлдардан қайта құрудың қажеті жоқ, егер сіздің жүйеңізде пакет орнатылмаған болса, бұл сіздің өміріңізді сақтайды.

Осы кезде сіз php54, php55, php56, php 70, php 71, php 72, php 73, php 74, php 80, php 81 немесе php 82-ді Port install пәрменімен орната аласыз, мысалы:

sudo port install php74
sudo port install php82

Белсенді PHP-ді ауыстыру үшін select пәрменін іске қосуға болады:

sudo port select --set php php82

phpbrew көмегімен PHP орнатыңыз

phpbrew-бұл PHP-дің бірнеше нұсқаларын орнатуға және басқаруға арналған құрал. Егер екі түрлі қолданба/жоба PHP-дің әртүрлі нұсқаларын қажет етсе және сіз виртуалды машиналарды пайдаланбасаңыз, бұл өте пайдалы болуы мүмкін.

Liip екілік орнатушысын пайдаланып PHP орнатыңыз

Тағы бір танымал нұсқа php-osx.liip.ch, ол 5.3-тен 7.3-ке дейінгі нұсқалар үшін бір жолды орнату әдістерін ұсынады. Ол Apple орнатқан PHP екілік файлдарын қайта жазбайды, бірақ бәрін бөлек жерде орнатады (/usr/local/php5).

Бастапқы кодтан құрастыру

Сіз орнатқан PHP нұсқасын басқаруға мүмкіндік беретін тағы бір нұсқа - оны өзіңіз құрастыру. Бұл жағдайда Apple әзірлеушілер орталығынан жүктелетін Xcode немесе Apple компаниясының “XCode үшін пәрмен жолы құралдары” алмастырғышы орнатылғанына көз жеткізіңіз.

All-in-One орнатушылар

Жоғарыда аталған шешімдер негізінен PHP-дің өзін өңдейді және Apache, Nginx немесе SQL server сияқты заттарды ұсынбайды. MAMP және XAMPP сияқты “All-in-one” шешімдер сіз үшін осы басқа бағдарламалық жасақтама компоненттерін орнатады және олардың барлығын біріктіреді, бірақ орнатудың қарапайымдылығы икемділікке байланысты.

Windows Орнату

Екілік файлдарды жүктеуге болады windows.php.net/download. PHP шығарылғаннан кейін PHP қалтасының түбіріне жол көрсету ұсынылады (мұнда php.EXE орналасқан), сондықтан сіз PHP-ді кез-келген жерден басқара аласыз.

Оқыту және жергілікті даму үшін Сіз PHP 5.4+ көмегімен кіріктірілген веб-серверді пайдалана аласыз, сондықтан оны орнату туралы алаңдамайсыз. Егер сізге толық веб-сервер мен MySQL кіретін" барлығы бір " қажет болса, онда XAMPP, EasyPHP, OpenServer және WAMP сияқты құралдар Windows даму ортасын жылдам іске қосуға көмектеседі. Дегенмен, бұл құралдар өндірістен сәл өзгеше болады, сондықтан Windows жүйесінде жұмыс істеп, Linux жүйесінде жұмыс істеп жатсаңыз, ортадағы айырмашылықтардан сақ болыңыз.

Егер сізге Windows жүйесінде өндірістік жүйені іске қосу қажет болса, онда IIS7 сізге ең тұрақты және жоғары өнімділікті қамтамасыз етеді. PHP орнатуды және басқаруды жеңілдету үшін phpmanager (IIS 7 үшін графикалық плагин) пайдалануға болады. IIS7 кіріктірілген FastCGI-мен бірге келеді және жұмыс істеуге дайын, тек PHP-ді өңдеуші ретінде орнату керек. Қолдау және қосымша ресурстар үшін арнайы аймақ бар iis.net PHP үшін.

Әдетте, сіздің қосымшаңызды әр түрлі даму және өндіріс орталарында іске қосу нақты уақыт режимінде іске қосу кезінде таңқаларлық қателіктерге әкелуі мүмкін. Егер сіз Windows-та дамып, Linux-қа (немесе Windows-тан басқа) орналастырсаңыз, виртуалды машинаны пайдалануды қарастырған жөн.

Крис Танкерслиде Windows көмегімен PHP әзірлеу үшін қандай құралдарды қолданатыны туралы өте пайдалы блог жазбасы бар.

Linux Орнату

GNU/Linux дистрибутивтерінің көпшілігі ресми репозиторийлерден қол жетімді PHP-мен бірге келеді, бірақ бұл пакеттер әдетте қазіргі тұрақты нұсқадан сәл артта қалады. Мұндай дистрибутивтерде PHP-дің жаңа нұсқаларын алудың көптеген жолдары бар. Мысалы, Ubuntu және Debian негізіндегі GNU / Linux дистрибутивтерінде Жергілікті пакеттерге арналған ең жақсы баламаларды Ондрей Сури Ubuntu-дағы жеке пакет мұрағаты (PPA) және Debian-дағы DPA/bikeshed арқылы ұсынады және қолдайды. Төменде олардың әрқайсысына арналған нұсқаулар берілген. Дегенмен, сіз әрқашан контейнерлерді пайдалана аласыз, PHP бастапқы кодын құрастыра аласыз және т. б.

Ubuntu негізіндегі дистрибутивтер

Ubuntu дистрибутивтері үшін Andrej SURY PPA көптеген PECL кеңейтімдерімен бірге PHP-дің қолдау көрсетілетін нұсқаларын ұсынады. Бұл PPA-ны жүйеге қосу үшін терминалда мына қадамдарды орындаңыз:

  1. Алдымен пәрменді пайдаланып жүйенің бастапқы бағдарламалық файлдарына PPA қосыңыз:
sudo add-apt-repository ppa:ondrej/php
  1. PPA қосқаннан кейін жүйенің пакеттер тізімін жаңартыңыз:
sudo apt update

Бұл сіздің жүйеңіздің PPA-да қол жетімді соңғы PHP пакеттеріне қол жеткізіп, оларды орната алатындығына кепілдік береді.

Debian негізіндегі дистрибутивтер

Debian негізіндегі дистрибутивтер үшін Ondrej Sukup сонымен қатар bikeshed (Debian-дағы PA аналогы) ұсынады. Жүйеге велосипед тұрағын қосу және жаңарту үшін мына қадамдарды орындаңыз:

  1. Сізде түбірлік қатынас бар екеніне көз жеткізіңіз. Олай болмаса, келесі командалар үшін sudo пайдалану қажет болуы мүмкін.
  2. Жүйе бумаларының тізімін жаңартыңыз:
sudo apt-get update
  1. lsb-release, ca сертификаттарын және curl орнатыңыз:
sudo apt-get -y install lsb-release ca-certificates curl
  1. Репозиторий үшін қолтаңба кілтін жүктеп алыңыз:
sudo curl -sSLo /usr/share/keyrings/deb.sury.org-php.gpg https://packages.sury.org/php/apt.gpg
  1. Жүйенің бағдарламалық жасақтамасының бастапқы кодтарына репозиторий қосыңыз:
sudo sh -c 'echo "deb [signed-by=/usr/share/keyrings/deb.sury.org-php.gpg] https://packages.sury.org/php/ $(lsb_release -sc) main" > /etc/apt/sources.list.d/php.list'
  1. Соңында, жүйенің пакеттер тізімін қайтадан жаңартыңыз:
sudo apt-get update

Осы қадамдарды орындау арқылы сіздің жүйеңіз PHP пакеттерінің соңғы нұсқаларын bikeshed-тен орната алады.

Каталогтардың жалпы құрылымы

Интернетке арналған бағдарламалар жаза бастағандар арасында жиі кездесетін сұрақ: "Мен өз материалдарымды қайда қоямын?"Көптеген жылдар бойы бұл жауап үнемі "құжаттың түбірі қайда."Бұл жауап толық болмаса да, бұл бастау үшін тамаша орын.

Қауіпсіздік мақсатында конфигурация файлдары сайтқа кірушілерге қол жетімді болмауы керек; сондықтан жалпыға қол жетімді сценарийлер жалпыға қол жетімді каталогта сақталады, ал жеке конфигурациялар мен деректер сол каталогтан тыс сақталады.

Сіз жұмыс істейтін әрбір команда, CMS немесе жақтау үшін осы нысандардың әрқайсысы стандартты каталог құрылымын пайдаланады. Алайда, егер біреу жобаны жалғыз бастаса, файлдық жүйенің қандай құрылымын қолдану керектігін білу қиын болуы мүмкін.

Пол М. Джонс PHP саласындағы ондаған мың github жобаларының кең таралған тәжірибелері туралы бірнеше фантастикалық зерттеулер жүргізді. Осы зерттеу негізінде ол стандартты файлдар мен каталог құрылымын, PHP пакетінің стандартты қаңқасын құрастырды. Бұл каталог құрылымында DocumentRoot public/, модульдік тесттер tests / каталогында болуы керек, ал Composer орнатқан үшінші тарап кітапханалары vendor/ каталогына жатады. Басқа файлдар мен каталогтар үшін PHP пакетінің стандартты жақтауын сақтау жобаға қатысушылар үшін ең мағыналы болады.
