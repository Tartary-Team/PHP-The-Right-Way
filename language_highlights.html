Бағдарламалау парадигмалары

PHP-көптеген бағдарламалау әдістерін қолдайтын икемді, динамикалық тіл. Осы жылдар ішінде ол айтарлықтай дамыды, атап айтқанда, PHP 5.0 (2004), PHP 5.3 (2009) және PHP 5.4 (2012) ішіндегі анонимді функциялар мен имен кеңістігіне сенімді объектіге бағытталған модель қосылды.

Объектіге бағытталған бағдарламалау (OOP)

PHP объектіге бағытталған бағдарламалау мүмкіндіктерінің өте толық жиынтығына ие, соның ішінде сыныптарды, дерексіз сыныптарды, интерфейстерді, мұрагерлікті, конструкторларды, клондауды, ерекшеліктерді және т.б. қолдау.

  • Объектіге бағытталған PHP туралы оқыңыз
  • Жеке қасиеттер туралы оқыңыз

Функционалды бағдарламалау

PHP бірінші дәрежелі функцияларды қолдайды, яғни функция айнымалыға тағайындалуы мүмкін. Айнымалы пайдаланушы немесе кірістірілген мүмкіндіктерге сілтеме жасай алады және динамикалық түрде шақырылады. Функциялар аргумент ретінде басқа функцияларға берілуі мүмкін (жоғары ретті функциялар деп аталатын функция) және функциялар басқа функцияларды қайтара алады.

Рекурсия, функцияның өзін-өзі шақыруына мүмкіндік беретін функцияны тіл қолдайды, бірақ PHP кодының көп бөлігі итерацияға бағытталған.

PHP 5.3 (2009) бастап жаңа анонимді функциялар (тұйықталуды қолдайды) бар.

PHP 5.4 объектінің көріну аймағына тұйықталу мүмкіндігін қосады, сонымен қатар шақырылатын объектілерді қолдауды жақсартады, осылайша оларды барлық жағдайларда дерлік анонимді функциялармен алмастыруға болады.

  • PHP-де функционалды бағдарламалау туралы толығырақ оқыңыз
  • Анонимді мүмкіндіктер туралы оқыңыз
  • Жабу сыныбы туралы оқыңыз
  • Толығырақ ақпарат жабылу бойынша RFC де берілген
  • Шақырылатын элементтер туралы оқыңыз
  • Call_user_func_array()көмегімен функцияларды динамикалық шақыру туралы оқыңыз

Метапрограммалау

PHP reflection API және Magic Methods сияқты механизмдер арқылы метапрограммалаудың әртүрлі формаларын қолдайды. __Get (), __set (), __clone (), __toString (), __invoke() және т. б. сияқты көптеген сиқырлы әдістер бар. бұл әзірлеушілерге сыныптың мінез-құлқына қосылуға мүмкіндік береді. Ruby әзірлеушілері PHP-де method_missing жоқ деп жиі айтады, бірақ ол __call() және __Call Static () ретінде қол жетімді.

  • Сиқырлы әдістер туралы оқыңыз
  • Рефлексия туралы оқыңыз
  • Шамадан тыс жүктеме туралы оқыңыз

Атаулардағы Бос орындар (Namespaces)

Жоғарыда айтылғандай, PHP қауымдастығында көптеген кодтар жасайтын көптеген әзірлеушілер бар. Бұл дегеніміз, бір кітапхананың PHP коды басқасымен бірдей сынып атауын қолдана алады. Екі кітапхана да бір имен кеңістігінде қолданылған кезде, олар соқтығысып, қиындықтар тудырады.

namespaces бұл мәселені шешеді. PHP анықтамалық нұсқаулығында сипатталғандай, имен кеңістігін имен кеңістігі файлдары сақталатын амалдық жүйенің каталогтарымен салыстыруға болады; бірдей атаумен екі файл бөлек каталогтарда қатар өмір сүре алады. Сол сияқты, бірдей атаумен екі PHP класы жеке PHP имен кеңістігінде қатар өмір сүре алады. Барлығы өте қарапайым.

Сіз үшін кодты басқа кітапханалармен соқтығысудан қорықпай, басқа әзірлеушілер пайдалана алатындай етіп имен кеңістігіне орналастыру маңызды.

Имен кеңістігін пайдаланудың ұсынылған әдістерінің бірі PSR-4 бөлімінде сипатталған, оның мақсаты кодты қосуға және ойнатуға мүмкіндік беретін стандартты файлдар, сыныптар және имен кеңістігі конвенциясын қамтамасыз ету болып табылады.

2014 жылдың қазан айында PHP-FIG алдыңғы іске қосу стандартын жойды: PSR-0. PSR-0 және PSR-4 екеуі де әлі де қолдануға жарамды. Соңғысы PHP 5.3-ті қажет етеді, сондықтан тек PHP 5.2-ді қолдайтын көптеген жобалар PSR-0-ді жүзеге асырады.

Егер сіз жаңа қолданба немесе пакет үшін іске қосу стандартын қолданғыңыз келсе, PSR-4-ті тексеріңіз.

  • Namespaces туралы оқыңыз
  • PSR-0 туралы оқыңыз
  • PSR-4 туралы оқыңыз

PHP стандартты кітапханасы

Стандартты PHP кітапханасы (SPL) PHP-мен бірге келеді және сыныптар мен интерфейстер жиынтығын ұсынады. Ол негізінен деректер құрылымдарының (стек, кезек, үйінді және т.б.) және осы деректер құрылымдарында қозғала алатын итераторлардан немесе SPL интерфейстерін жүзеге асыратын өз сыныптарыңыздан тұрады.

  • SPL туралы оқыңыз

Пәрмен жолы интерфейсі (CLI)

PHP веб-қосымшаларды жазу үшін жасалған, бірақ сонымен бірге командалық жол интерфейсі (CLI) бағдарламаларын жазу үшін пайдалы. PHP пәрмен жолы бағдарламалары тестілеу, орналастыру және қолданбаларды басқару сияқты жалпы тапсырмаларды автоматтандыруға көмектеседі.

CLI PHP бағдарламалары қуатты, өйткені сіз веб-интерфейсті құрудың және қорғаудың қажеті жоқ қолданба кодын тікелей пайдалана аласыз. Тек CLI PHP сценарийлерін Жалпыға Ортақ веб-түбірге орналастырмағаныңызға көз жеткізіңіз!

PHP пәрмен жолынан іске қосып көріңіз:

> php -i

‘-i’ параметрі phpinfo() функциясы сияқты PHP конфигурациясын шығарады.

‘-a’ параметрі ruby немесе python интерактивті қабығындағы IRB-ге ұқсас интерактивті қабықты ұсынады. Пәрмен жолының басқа да пайдалы нұсқалары бар.

“Hello, $name” қарапайым CLI бағдарламасын жазайық. Мұны байқап көру үшін hello.php деп аталатын файл жасаңыз.төменде көрсетілгендей.

<?php
if ($argc !== 2) {
    echo "Usage: php hello.php <name>" . PHP_EOL;
    exit(1);
}
$name = $argv[1];
echo "Hello, $name" . PHP_EOL;

PHP сіздің сценарийіңіз іске қосылатын дәлелдерге негізделген екі арнайы айнымалыны орнатады. $argc-аргументтер санын қамтитын бүтін айнымалы, ал каждого argv - әрбір Аргументтің мәнін қамтитын массив айнымалысы. Бірінші дәлел әрқашан PHP сценарийінің Файл атауы болып табылады, бұл жағдайда hello.php.

exit() өрнегі командалық қабыққа команданың орындалмағаны туралы хабарлау үшін нөлдік емес санмен қолданылады. Жиі қолданылатын шығу кодтарын мына жерден табуға болады.

To run our script, above, from the command line:

> php hello.php
Usage: php hello.php <name>
> php hello.php world
Hello, world

Xdebug

Бағдарламалық жасақтаманы әзірлеудегі ең пайдалы құралдардың бірі-дұрыс отладчик. Бұл кодтың орындалуын бақылауға және стек мазмұнын бақылауға мүмкіндік береді. PHP түзеткіші Xdebug-ті тоқтату нүктелерін қамтамасыз ету және стекті тексеру үшін әртүрлі IDE қолдана алады. Бұл сонымен қатар PHPUnit және KCachegrind сияқты құралдарға кодты қамтуды талдауға және кодты профильдеуге мүмкіндік береді.

Егер сіз ‘var_dump()/print_r()’ - ге жүгінгіңіз келсе және әлі де шешім таба алмасаңыз-сізге түзету құралын пайдалану қажет болуы мүмкін.

Xdebug орнату қиын болуы мүмкін, бірақ оның маңызды мүмкіндіктерінің бірі - “қашықтан жөндеу” - Егер сіз кодты жергілікті түрде жасап, содан кейін оны виртуалды машинаның ішінде немесе басқа серверде сынап көрсеңіз, қашықтан жөндеу-бұл сіз бірден қосқыңыз келетін мүмкіндік.

Дәстүр бойынша Сіз Apache VHost-ты өзгертесіз немесе. осы мәндері бар .htaccess файлы:

php_value xdebug.remote_host 192.168.?.?
php_value xdebug.remote_port 9000

“remote host” және “remote port” жергілікті компьютерге және IDE-ді тыңдау үшін конфигурациялаған портқа сәйкес келеді. Содан кейін IDE-ді “қосылымдарды тыңдау” режиміне қойып, URL мекенжайын жүктеу ғана қалады:

http://your-website.example.com/index.php?XDEBUG_SESSION_START=1

Енді сіздің ide ортаңыз сценарий орындалған кезде ағымдағы күйді ұстап қалады, бұл сізге тоқтау нүктелерін орнатуға және жадтағы мәндерді тексеруге мүмкіндік береді.

Графикалық отладчиктер кодты кезең-кезеңімен орындауды, айнымалыларды тексеруді және кодты нақты уақыт режимінде бағалауды өте жеңілдетеді. Көптеген IDE-де xdebug көмегімен кірістірілген немесе плагинге негізделген графикалық жөндеу қолдауы бар. MacGDBp-бұл macOS үшін ақысыз, дербес, ашық бастапқы xdebug графикалық интерфейсі.

  • Xdebug туралы көбірек біліңіз
  • MacGDBp туралы көбірек біліңіз
