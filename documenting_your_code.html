Кодты документтелу

PHPDoc

PHPDoc PHP кодты түсіндіру үшін көмекші байланыс. Оларды тізімдейтін анықтамалар жататын. Толық тізімі мен мысалдарды PHPDoc бойынша көмек кітапшасында таба аласыз.

Төменде сізге кейбір әдістермен бір класты документтелу жолымен мысал берілген:

<?php
/**
 * @author A Name <a.name@example.com>
 * @link https://www.phpdoc.org/docs/latest/index.html
 */
class DateTimeHelper
{
    /**
     * @param mixed $anything \DateTime объектіне айналдыратын нәрсе
     *
     * @throws \InvalidArgumentException
     *
     * @return \DateTime
     */
    public function dateTimeFromAnything($anything)
    {
        $type = gettype($anything);

        switch ($type) {
            // \DateTime объектіне айналдыру үшін көмекші код
        }

        throw new \InvalidArgumentException(
            "'{$type}' түріндегі параметрді \DateTime объектіне айналдыру сәтсіз болды"
        );
    }

    /**
     * @param mixed $date \DateTime объектіне айналдыратын нәрсе
     *
     * @return void
     */
    public function printISO8601Date($date)
    {
        echo $this->dateTimeFromAnything($date)->format('c');
    }

    /**
     * @param mixed $date \DateTime объектіне айналдыратын нәрсе
     */
    public function printRFC2822Date($date)
    {
        echo $this->dateTimeFromAnything($date)->format('r');
    }
}

Класс бойынша барлық үшін барлық бағдарлама @author тегі мен @link тегін қамтиды. @author тегі кодтың авторын анықтау үшін пайдаланылады және бірнеше авторларды анықтау үшін кейбір ретпен жасалуы мүмкін. @link тегі код пен веб-сайт арасындағы байланысты көрсету үшін пайдаланылады.

Сынып ішінде бірінші әдіс @param тегінінде өтінімге берілетін параметрдің түрін, атын және сипаттамасын анықтау үшін пайдаланылады. Қоса, олда @return мен @throws тегтері бар; сияқты, келесі түйіндемелерді документтеу үшін пайдаланылады.

Екінші және үшінші әдістер өте салыстырылуы мүмкін және бір @param тегі бар болатын бірінші әдістен бірақ кездейсоқ. Екінші және үшінші әдістердің документтегінін орналасуының маңызды айырымы екінші мен үшінші әдістердің документтегінің @return тегінін қосу/кестеу болады. @return void екінші жолдасқан және үшінші әдістерді документтегіні қосбасыздан қалдыруда (біріктіруде) тарихи себепті, бір түрде ешқандай қайтару жоқ (қайтару жоқ) дегенін білдіреді.
